# Governance

Visor is an open-source project maintained by [Low Orbit Studio](https://loworbit.studio).

## Maintainers

See [MAINTAINERS.md](MAINTAINERS.md) for the current list of maintainers and their areas of ownership.

## Decision Making

Visor is currently a **solo-maintainer project**. The lead maintainer has final say on direction, scope, and releases. Decisions are made in the open — significant changes are discussed in issues or Discussions before implementation begins.

As the contributor community grows, governance will evolve toward a consensus-based model.

## Contributions

All contributions are welcome. See [CONTRIBUTING.md](CONTRIBUTING.md) for the full process. The short version:

- Bug fixes and small improvements: open a PR directly.
- New components or significant API changes: open an issue to align on approach first.
- Breaking changes: always discuss before investing in implementation.

## Releases

Visor uses [Changesets](https://github.com/changesets/changesets) for versioning and publishing. Releases are cut from `main` when there's meaningful new content. There is no fixed cadence.

## Code of Conduct

All participants are expected to follow the project's [Code of Conduct](CODE_OF_CONDUCT.md).
