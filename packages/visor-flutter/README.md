# visor_core

Flutter tokens and theme builder for the [Visor design system](https://visor.design).

## Two-Layer Model

Visor's Flutter integration uses the same two-layer model as its React side:

1. **`visor_core` (this package)** — `ThemeExtension` classes and the `VisorTheme.build()` builder. Semver-managed, updates propagate via `flutter pub upgrade`.
2. **Generated token files** — Concrete color, typography, spacing, radius, shadow, and motion values. Generated by the Visor CLI from your project's `.visor.yaml` and written into your project's `packages/ui/lib/src/` tree. You own and can edit these files.

## Usage

```dart
import 'package:flutter/material.dart';
import 'package:visor_core/visor_core.dart';
import 'package:ui/ui.dart'; // your generated tokens

void main() {
  runApp(
    MaterialApp(
      theme: VisorAppTheme.light,
      darkTheme: VisorAppTheme.dark,
      home: const HomePage(),
    ),
  );
}
```

In widgets, access tokens via `Theme.of(context).extension<VisorColors>()` or through the Material `ColorScheme` / `TextTheme` that `VisorTheme.build()` populated from your tokens.

## Generating tokens

```bash
visor theme apply ./my-brand.visor.yaml \
  --adapter flutter \
  --output ./packages/ui
```

See the [Visor Flutter docs](https://visor.design/docs/flutter) for the full workflow.
