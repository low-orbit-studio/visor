## Open/Close Delay

Control how quickly the card appears and disappears with `openDelay` and `closeDelay` (in milliseconds) on the root `HoverCard` component.

```tsx
<HoverCard openDelay={200} closeDelay={100}>
  <HoverCardTrigger asChild>
    <a href="#">Hover me</a>
  </HoverCardTrigger>
  <HoverCardContent>
    <p>Appears after 200ms, disappears after 100ms.</p>
  </HoverCardContent>
</HoverCard>
```

## Installation

```bash
npx visor add hover-card
```

This copies two files into your project:

- `components/ui/hover-card/hover-card.tsx` — the component
- `components/ui/hover-card/hover-card.module.css` — the styles

## Usage

```tsx
import {
  HoverCard,
  HoverCardTrigger,
  HoverCardContent,
} from '@/components/ui/hover-card/hover-card';

export default function Example() {
  return (
    <HoverCard>
      <HoverCardTrigger asChild>
        <a href="#">Hover me</a>
      </HoverCardTrigger>
      <HoverCardContent>
        <p>Card content here.</p>
      </HoverCardContent>
    </HoverCard>
  );
}
```

## Sub-components

| Component | Element | Purpose |
|-----------|---------|---------|
| `HoverCard` | Root | Context provider (wraps Radix HoverCard.Root) |
| `HoverCardTrigger` | Any element | The element that triggers the card on hover |
| `HoverCardContent` | Floating panel | The card container; accepts `side`, `align`, and `sideOffset` |

## Accessibility

- Built on [Radix UI HoverCard](https://www.radix-ui.com/docs/primitives/components/hover-card) — the content region is not focusable by default. Use HoverCard for supplemental, non-critical information only.
- For content that must be keyboard-accessible (e.g., links, buttons inside the card), use [Popover](/docs/components/overlay/popover) instead.
- The trigger should be a focusable element (link or button) so keyboard users can navigate to the associated content in another way.
- Screen readers are not guaranteed to announce hover card content; treat it as decorative preview text.
